# תכונת ניהול העלאות למנהל

## סקירה כללית
המנהל יכול כעת לראות את כל הקבצים שהמשתמשים העלו בדף שליחת קובץ, כולל אפשרות להוריד ולצפות בקבצים.

## מה נוסף?

### 1. טאב חדש בדף הניהול
- טאב "העלאות" חדש בדף הניהול (`/admin`)
- מציג את מספר ההעלאות הממתינות בכותרת הטאב
- מציג את כל ההעלאות עם פרטים מלאים

### 2. פרטי העלאה
כל העלאה מציגה:
- שם הספר
- שם המשתמש שהעלה
- שם הקובץ המקורי
- גודל הקובץ
- מספר שורות
- תאריך ושעת העלאה
- סטטוס (ממתין/אושר/נדחה)

### 3. פעולות זמינות
- **הורד קובץ** - הורדה ישירה של הקובץ
- **צפה בקובץ** - פתיחת הקובץ בטאב חדש
- **אשר** - אישור ההעלאה (רק להעלאות ממתינות)
- **דחה** - דחיית ההעלאה (רק להעלאות ממתינות)

### 4. סטטיסטיקה
- קארד חדש בראש דף הניהול מציג את מספר ההעלאות הכולל
- מציג את מספר ההעלאות הממתינות באדום

## קבצים שנוצרו/עודכנו

### קבצים חדשים:
1. `src/app/api/admin/uploads/list/route.js` - API לקבלת רשימת העלאות עם URLs
2. `src/app/api/admin/uploads/update-status/route.js` - API לעדכון סטטוס העלאה

### קבצים שעודכנו:
1. `src/app/admin/AdminClient.jsx` - הוספת טאב העלאות וממשק משתמש

## איך זה עובד?

1. משתמש מעלה קובץ בדף `/upload`
2. הקובץ נשמר ב-Vercel Blob Storage תחת `dev/data/uploads/`
3. מטא-דאטה נשמרת ב-`data/uploads-meta.json`
4. המנהל רואה את ההעלאה בדף הניהול
5. המנהל יכול להוריד/לצפות בקובץ ולאשר או לדחות אותו

## הערות טכניות

- הקבצים מאוחסנים ב-Vercel Blob Storage
- ה-URLs נוצרים דינמית מה-Blob Storage
- הסטטוס נשמר במטא-דאטה ומעודכן בזמן אמת
- המערכת תומכת בקבצי TXT בלבד (כפי שהוגדר בדף ההעלאה)
